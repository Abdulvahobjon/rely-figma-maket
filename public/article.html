Ассортимент продуктового ассортимента

Обратный заказ

Обратный заказ – это сценарий заказа товаров при котором решение о том, сколько поставлять товаров в магазин принимает поставщик.
Поставщик, при таком сценарии заказа, самостоятельно анализирует остатки, отвечает за размер поставки и порчу остатков, если она произошла. В ряде случаев товар может отгружаться «на реализацию», то есть магазин оплачивает только за проданные товары, а приход оформляется как приход на ответственное хранение.
Для оперативного контроля остатков для товаров с коротким сроком годности поставщику очень важно знать текущие остатки товаров в магазине и своевременно их пополнять. Сервис 1С-Товары позволяет задать для товара основного поставщика и, если товару установлен сценарий заказа «Обратный заказ», поставщику становиться доступна информация об остатках таких товаров на момент последней выгрузки данных из магазина.
Сервис 1С-Товары позволяет синхронизировать наименования товаров у поставщика и магазина, что дает возможность не тратить время на отправку заявок и ввод приходных документов от поставщика.

Обратный заказ у магазина

Прежде всего для группы товаров необходимо установить сценарий заказа «Обратный заказ» и сохранить настройки.

Второе обязательно действие – для всех товаров установить основного поставщика. Делается это на закладке «Работа с поставщиками»
Для установки поставщика необходимо:
Выбрать группу товаров, добавить ее в табличную часть. Если в списке есть товары, по которым вы не готовы установить основного поставщика – такой товар нужно удалить из списка.
По кнопке заполнить вы можете загрузить список поставщиков, которые ранее поставляли товары данной товарной группы и сравнить цены поставщиков.
При необходимости можно добавить нового поставщика и загрузить для сравнения цены его прайс листа (подробнее об этом в отдельном видео)

Из списка поставщиков необходимо выбрать основного поставщика для списка товаров, который открыт на экране и установить его для всех товаров. Очень важно – у поставщика должен быть заполнен ИНН и КПП. Обмен документами между организациями без ИНН в сервисе 1С-Товары не возможен.

После сохранения информация об остатках товаров, для которых был установлен сценарий «Обратный заказ» будет доступна поставщику, установленному как основной.

Обратный заказ у магазина

Возможности модуля сервиса 1С-Товары определяется набор подключенных тарифов на портале 1С. Если для компании подключен тариф поставщика, то в обработке будет доступна закладка «Функционал поставщика». На закладке «Функционал поставщика» в разделе «обратный заказ» можно получить список магазинов, которые предоставили информацию о своих остатках.

Выбрав магазин можно увидеть список товаров магазина, остатки и дату актуальности данных, а также настройки минимального остатка и партии заказа, которые установил магазин для каждого товара.

Сервис 1С-Товары автоматически рассчитает рекомендованный размер заказа. При необходимости его можно исправить. Сервис 1С-Товары выполняет синхронизацию справочников товаров между поставщиком и магазином по штрих коду и по полному соответствию наименований.
Если справочники не сопоставились автоматически – соответствие можно задать вручную. Установленные соответствия позволяют в дальнейшем автоматически загружать заявки в базу поставщика и отправлять накладные в магазин. Магазину не нужно будет набирать накладную для того, чтобы принять товар.
При сохранении заявки в системе будет создан типовой документ Заказ клиента, с которым можно продолжить работать по стандартной технологии типовой конфигурации.
